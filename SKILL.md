# Literature Single-Paper Decompose

把一篇論文整理成「**每個論斷都能 grep 回原文**」的理論建構分析與架構圖。

**鐵律**：本 skill 描述「**本文如何使用文獻／如何建構理論**」（citation context，有本文內文支撐），**永不**宣稱「**原典實際主張什麼**」（那要讀原典，是後續工作）。守住這條線，幻覺風險在定義上就極低。

資料規格與受控詞彙見 `references/schema-contract.md`。本檔給流程與判準。

## 核心防幻覺原則（為什麼）

- **citation context ≠ theory claim**：「本文如何用 X」≠「X 實際主張什麼」。只做前者。
- **存在性 ≠ 正確性**：查到文獻存在只證明它存在，不證明轉述它的理論正確。
- **本文明示 vs 分析者詮釋**：兩者必須分開標記（見 C2）。最隱蔽的幻覺不是捏造原典，而是**過度詮釋被分析文本**（把自己的歸納當成本文原話）。
- **可回溯 = 可稽核**：每條論斷附逐字引文且能在母本字串搜尋到，才算數。
- **標籤誠實**：理論關係圖的邊必須對應引文；揭露查證分母，不可讓「部分查證」看起來像「全部」。

## 路徑配置（可攜）

本 skill 用兩個可配置路徑（見 repo `README.md`）；放到新專案/新機器時依需要調整：
- **`WORK_DIR`**（工作產物）：預設當前專案下的 `./ltm-work/`，或使用者指定的目錄。
- **`KB_DIR`**（可選的長期知識庫根）：預設使用者的 Obsidian vault 下 `literature-kb/`；未設定就略過 KB 卡片化收尾。

下文出現 `WORK_DIR/`、`KB_DIR/` 即指這兩個路徑；動工前若使用者未指定，先用預設值並在回報中說明落腳處。

## 產出

工作產物存 `WORK_DIR/<citekey>*.md`（母本 + 各步報告，或合併為 `<citekey>__digest.md`）。
> 可選收尾：若使用者要累積長期知識網絡，把成果轉成 Obsidian KB 卡片（CI/TC 等）寫入 `KB_DIR`，欄位與資料夾依 `references/schema-contract.md`。預設**不**自動做，使用者要求才做。

## 第 1 步 — 匯入（擇優擷取，不可盲用單一工具）

PDF → 乾淨 Markdown 母本（`WORK_DIR/<citekey>.md`），是後續所有逐字引文（C1/C3）的 grep 回溯來源。

1. 預設 **markitdown**（`markitdown <pdf> -o <out>`）：保留標題/結構，多數 PDF 表現好。
2. **品質檢查（必做）**：抽幾個已知片語，檢查有無①浮水印雜訊（重複字母）②**系統性吞空格**（如 `Logit,probit,rescaling` 黏成一團）。空格遺失明顯即判定失效。
3. **失效則 fallback `pypdf`**（`python3 + pypdf`，逐頁加 `===== PDF page N =====`）。對部分 SAGE/舊版式 PDF 空格保留更好，頁碼可作定位。
4. **公式限制**：公式重的論文兩工具都無法可靠擷取符號 → C1 逐字引文**只適用散文句**，公式類主張改概念描述並標「公式擷取受限」，不假裝有逐字依據。

> 為何重要：母本吞了空格，逐字引文就 grep 不到、防幻覺機制失效——匯入品質是整條防線的地基。

## 第 2 步 — 資訊整理（APA7 + 結構化摘要 + 理論相關文獻查證）

1. **文章 APA7**：完整引用 + DOI + 期刊分級（JCR quartile / SSCI·TSSCI·AHCI / IF）。查不到的欄位標 `unknown`，**不捏造**。APA7 格式規範不確定時參考使用者既有慣例。
2. **結構化重點摘要**（非成段翻譯）：條列「背景 / 研究問題 / 核心論點 / 研究方法 / 主要發現 / 貢獻」。關鍵主張盡量帶母本定位。
3. **理論相關參考文獻**：只挑承載理論的文獻（對應後述 `ci_layer ∈ {foundation, definition, mechanism, counter}`），轉 APA7 + 連網查證：
   - `firecrawl_search`（主，以「標題+第一作者+年份」）＋必要時 Scholar Gateway `semanticSearch`。
   - 比對 DOI/期刊/卷期頁，標 ✅ 已驗證 / ❓ 存疑 / ❌ 查無；查 JCR 與收錄。
   - **揭露分母**：文獻多時只查核心子集，須明講「全文 N 筆、核心查 M 筆、餘未逐筆查證」，不可讓部分查證看似全部。
   - ⚠️ 留意書目瑕疵：卷期號誤標、或 AI 輔助寫作誤貼連結（如某筆參考的 URL 是 `chatgpt.com/...` 對話連結）→ 標存疑並在回報點出。查證常能順手抓到這類錯誤，是附加價值。

## 第 3 步 — 理論建構釐清（核心，套用 C1–C4）

拆解作者**如何建構理論**，分三層：
- **L1 理論來源**：本文借用哪些既有理論/概念，出自哪些文獻。
- **L2 本文操作**：作者自己的理論動作（重新切割、操作化、二分等）。
- **L3 理論關係**：本文如何與各理論對話。

**每條論斷必須滿足 C1–C4**（這是本 skill 的命脈）：
- **C1 逐字引文**：附被分析文的 `verbatim_quote`（原文逐字），不可只給頁碼或改寫。
- **C2 evidence_type**：標 `explicit`（本文有句子直接表達，附引文）或 `interpreted`（分析者歸納的分類/角色標籤，如 foundation/counter 群組）。誠實分開，`interpreted` 不得冒充本文原話。
- **C3 可回溯定位**：用母本**行號或可字串搜尋的引文**定位，確保任何人能 grep 回原文核對。
- **C4 受控關係詞彙＋貼近本文用語**：關係只能用 `draws-on / defines / applies / extends / challenges / repositions / controls-for`，且優先採本文自己的動詞——本文寫 "control for" 就標 `controls-for`，**不准**升級成語意更強的詞（如「統計控制」寫成「理論調和」）。

> 自我檢查：完成後隨機挑 2–3 條論斷，實際 `grep` 引文回母本。grep 不到 = 定位幻覺或母本品質問題，必須修。

## 第 4 步 — 架構圖

把「**理論來源(L1) ↔ 本文理論架構(L2) ↔ 研究方法 ↔ 研究發現**」整合成 Mermaid `graph TD`（分 subgraph 四區），並附 **邊—證據對照表**（每條理論邊 ↔ relation 詞彙 ↔ 逐字引文 ↔ 母本定位 ↔ evidence_type）。
- 圖中**每條理論關係邊**都要能回溯到第 3 步的引文；研究發現節點來自摘要的本文結果。圖**不引入**未驗證的新主張。
- 用 `validate_and_render_mermaid_diagram` 驗證語法（label 含 `&` 須在引號內）。
- 標籤誠實：這是「本文理論架構圖」（基於 citation context），**不是**「已驗證的原典理論關係圖」。

## 收尾回報

**先結論後細節**：APA7、摘要要點、查證統計（✅/❓/❌ 各幾筆＋分母）、可疑書目、理論骨架一句話、架構圖位置。

## Forbidden Actions（負面表列）

1. 把論斷寫成「某文獻主張 X」——只能寫「本文用某文獻作為 [角色]／本文如何與之對話」。
2. 把存在性查證當成理論主張正確的證據。
3. 描述理論關係/引用脈絡時不附本文逐字引文，或附了引文卻無法在母本字串搜尋到（違反 C1/C3）。
4. 把分析者歸納的關係或角色標籤標成 `explicit`、或冒充本文原話（違反 C2）——歸納一律標 `interpreted`。
5. 用受控集合以外的關係詞彙，或把本文弱動詞升級成更強的詞（違反 C4，例：control for → 調和）。
6. 公式類主張假裝有逐字依據（母本擷取不全時）——須標「公式擷取受限」。
7. 未經查證對 reference 標「已驗證」；捏造 DOI／期刊分級——查不到一律 `unknown`。
8. 宣稱「已查證全部參考文獻」而實際只查核心子集——必須揭露分母。
9. 把本文架構圖命名為「已驗證理論關係圖 / Confirmed Theory Graph」——它基於 citation context，須誠實標示。
