# Literature Single-Paper Decompose

> **作者 / Author**：Wen-Cheng Lin　｜　**狀態：實驗性 skill（experimental）**
>
> ⚠️ **使用前請仔細考慮**。本 skill 以「降低幻覺」為設計目標,但**無法保證零幻覺**:查證仰賴外部搜尋、
> 自我 grep 為抽樣、最終輸出仍依模型遵循指令。產出僅供研究輔助,**重要引用與理論歸屬請務必自行覆核**。
> 是否採用由使用者自行評估與負責。
>
> ⚠️ **API 風險(務必理解)**:本 skill 的查證會呼叫外部服務——
> ① **隱私**:書目查證(`firecrawl`／學術搜尋)與選用的第 5 步 cross-AI 檢核,會把**論文片段／逐字引文外送**第三方;
> 未發表稿件請自行斟酌,必要時略過連網查證或外部檢核;
> ② **費用／額度**:相關 API 可能按量計費或有額度上限,成本由使用者承擔;
> ③ **金鑰**:任何 API key 屬個人機密,**切勿貼進對話框**,僅存本機環境變數。

把一篇論文整理成「**每個論斷都能 grep 回原文**」的理論建構分析與架構圖。

**鐵律**：本 skill 描述「**本文如何使用文獻／如何建構理論**」（citation context，有本文內文支撐），**永不**宣稱「**原典實際主張什麼**」（那要讀原典，是後續工作）。守住這條線，幻覺風險在定義上就極低。

資料規格與受控詞彙見 `references/schema-contract.md`。本檔給流程與判準。

## 核心防幻覺原則（為什麼）

- **citation context ≠ theory claim**：「本文如何用 X」≠「X 實際主張什麼」。只做前者。
- **存在性 ≠ 正確性**：查到文獻存在只證明它存在，不證明轉述它的理論正確。
- **本文明示 vs 分析者詮釋**：兩者必須分開標記（見 C2）。最隱蔽的幻覺不是捏造原典，而是**過度詮釋被分析文本**（把自己的歸納當成本文原話）。
- **可回溯 = 可稽核**：每條論斷附逐字引文且能在母本字串搜尋到，才算數。
- **標籤誠實**：理論關係圖的邊必須對應引文；揭露查證分母，不可讓「部分查證」看起來像「全部」。

## 路徑配置（可攜）

本 skill 用兩個可配置路徑（見 repo `README.md`）；放到新專案/新機器時依需要調整：
- **`WORK_DIR`**（工作產物）：預設當前專案下的 `./ltm-work/`，或使用者指定的目錄。
- **`KB_DIR`**（可選的長期知識庫根）：預設使用者的 Obsidian vault 下 `literature-kb/`；未設定就略過 KB 卡片化收尾。

下文出現 `WORK_DIR/`、`KB_DIR/` 即指這兩個路徑；動工前若使用者未指定，先用預設值並在回報中說明落腳處。

## 產出

工作產物存 `WORK_DIR/<citekey>*.md`（母本 + 各步報告，或合併為 `<citekey>__digest.md`）。

> **可讀產物須收斂到已驗證 artifact（P0-1，防「讀者讀到最弱產物」）**：散文式 `digest` 的每條 L1/L3 理論論斷後，必須內聯帶 `[claim-id · L<行號> · ✅/⚠️/🔴]`，狀態取自 `verify_claims.sh` 的全量驗證結果（見第 3 步）。digest 末尾附完整驗證表（claim-id ↔ 行號 ↔ 狀態）。**未經 verify_claims 驗證、或含 🔴MISS 的 digest 不得當作可信產物對外引用**——它只是導覽，可信度以驗證表為準。

> 可選收尾：若使用者要累積長期知識網絡，把成果轉成 Obsidian KB 卡片（CI/TC 等）寫入 `KB_DIR`，欄位與資料夾依 `references/schema-contract.md`。預設**不**自動做，使用者要求才做。

## 第 1 步 — 匯入（擇優擷取，不可盲用單一工具）

PDF → 乾淨 Markdown 母本（`WORK_DIR/<citekey>.md`），是後續所有逐字引文（C1/C3）的 grep 回溯來源。

1. 預設 **markitdown**（`markitdown <pdf> -o <out>`）：保留標題/結構，多數 PDF 表現好。
2. **品質檢查（必做）**：抽幾個已知片語，檢查有無①浮水印雜訊（重複字母）②**系統性吞空格**（如 `Logit,probit,rescaling` 黏成一團）。空格遺失明顯即判定失效。
   - **可攜指令**：直接跑 `bash scripts/biblio_healthcheck.sh <母本.md>`，一次涵蓋本步①②與第 2 步全文書目體檢。**重複字母偵測務必用該 script（走 perl backref）——不要臨場下 `grep '\1'`，本機 grep 可能是 ugrep，`\1` 不支援會靜默失效。** script 已內建排除 URL/DOI/`www` 的 guard，避免把 hex hash 誤報成浮水印。
3. **失效則 fallback `pypdf`**（`python3 + pypdf`，逐頁加 `===== PDF page N =====`）。對部分 SAGE/舊版式 PDF 空格保留更好，頁碼可作定位。
4. **公式限制**：公式重的論文兩工具都無法可靠擷取符號 → C1 逐字引文**只適用散文句**，公式類主張改概念描述並標「公式擷取受限」，不假裝有逐字依據。

> 為何重要：母本吞了空格，逐字引文就 grep 不到、防幻覺機制失效——匯入品質是整條防線的地基。

## 第 2 步 — 資訊整理（APA7 + 結構化摘要 + 理論相關文獻查證）

1. **文章 APA7**：完整引用 + DOI + 期刊分級（JCR quartile / SSCI·TSSCI·AHCI / IF）。查不到的欄位標 `unknown`，**不捏造**。APA7 格式規範不確定時參考使用者既有慣例。
2. **結構化重點摘要**（非成段翻譯）：條列「背景 / 研究問題 / 核心論點 / 研究方法 / 主要發現 / 貢獻」。關鍵主張盡量帶母本定位。
3. **理論相關參考文獻**：只挑承載理論的文獻（對應後述 `ci_layer ∈ {foundation, definition, mechanism, counter}`），轉 APA7 + 連網查證：
   - `firecrawl_search`（主，以「標題+第一作者+年份」）＋必要時 Scholar Gateway `semanticSearch`。
   - 比對 DOI/期刊/卷期頁，標 ✅ 已驗證 / ❓ 存疑 / ❌ 查無；查 JCR 與收錄。
   - **撤稿查證（必做）**：對核心文獻查撤稿/更正狀態（Retraction Watch、期刊撤稿/expression of concern 聲明）。命中撤稿醒目標 🚩 並在回報點出——引用已撤稿文獻支撐理論，是理論建構的硬傷。查不到撤稿紀錄只代表「未發現」，不等於保證未撤稿，據實標示。
   - **揭露分母**：文獻多時只查核心子集，須明講「全文 N 筆、核心查 M 筆、餘未逐筆查證」，不可讓部分查證看似全部。
4. **全文書目體檢（必做，便宜全掃，與第 3 點的深度查證分開）**：對**全部 N 筆** references 做一次低成本掃描，專抓不需連網就能發現的硬傷——不是逐筆連網查證，所以能涵蓋全文卻幾乎不增成本。為什麼獨立成一項：深度查證只挑理論子集（M 筆），而 AI 誤貼／typo 常落在子集外，靠運氣會漏；全掃讓這層附加價值**穩定觸發**。
   - **掃描樣式**：AI 輔助寫作誤貼的對話連結（`chatgpt.com/`、`claude.ai/`、`gemini.google.com/`、`bard.` 等）、DOI 拼錯（`doi.ogr`、缺 scheme、重複 `https`）、明顯失效或佔位 URL、卷期號/年份明顯矛盾。
   - **母本陷阱（重要）**：markitdown 常把 URL 拆成逐字加空格（如 `h t t p s : / / c h a t g p t . c o m`），直接 grep `chatgpt.com` 會漏判成「未發現」。掃描前先移除候選行空白再比對（例 `tr -d ' '` 後再 grep），否則會出現假陰性。
   - **可攜指令**：`bash scripts/biblio_healthcheck.sh <母本.md>` 已實作 A–E 全部樣式（去空格破陷阱、DOI typo、placeholder、浮水印、黏字），跨 ugrep/GNU/BSD 環境一致。**唯 [F] 卷期/年份矛盾無機制保證**——grep 無法語意比對（如 1912 文獻卻標 2019），須人工掃 references 年份/卷期是否互相矛盾，不可只依賴 script。
   - 命中標 ⚠️ 並在回報點出（含 ref 編號與問題）；查無則回報「全文 N 筆書目體檢未見明顯瑕疵」。誠實區分「全掃未見」與「逐筆已驗證」——前者只排除明顯硬傷，不等於每筆都對。

## 第 3 步 — 理論建構釐清（核心，套用 C1–C5）

拆解作者**如何建構理論**，分三層：
- **L1 理論來源**：本文借用哪些既有理論/概念，出自哪些文獻。
- **L2 本文操作**：作者自己的理論動作（重新切割、操作化、二分等）。
- **L3 理論關係**：本文如何與各理論對話。

**每條論斷必須滿足 C1–C5**（這是本 skill 的命脈）：
- **C1 逐字引文**：附被分析文的 `verbatim_quote`（原文逐字），不可只給頁碼或改寫。
- **C2 evidence_type**：標 `explicit`（本文有句子直接表達，附引文）或 `interpreted`（分析者歸納的分類/角色標籤，如 foundation/counter 群組）。誠實分開，`interpreted` 不得冒充本文原話。
- **C3 可回溯定位**：用母本**行號或可字串搜尋的引文**定位，確保任何人能 grep 回原文核對。
- **C4 受控關係詞彙＋貼近本文用語**：關係只能用 `draws-on / defines / applies / extends / challenges / repositions / controls-for`，且優先採本文自己的動詞——本文寫 "control for" 就標 `controls-for`，**不准**升級成語意更強的詞（如「統計控制」寫成「理論調和」）。
- **C5 引文—論斷適配性（防「可追溯 ≠ 可靠」）**：C1 引文必須**實際支撐**該論斷的關係，不只是「字串存在於母本」。最隱蔽的殘留幻覺是：引文確實 grep 得到，但它**支持的是別的東西**（相鄰句、背景鋪陳、他人觀點、被作者反對的立場），與論斷錯位。每條論斷須用一句話說明「此引文如何支撐此 relation」；若引文只證明「提到」卻被寫成「依賴/延伸」，即為**適配失敗**，須降級或改寫。涉及否定/讓步語氣（however / although / not / may / suggests）時，須保留該語氣，不得抹平成肯定主張。

> **自我檢查（全量，非抽樣）**：完成後對**每一條** C1 引文逐條 `grep` 回母本——用 `bash scripts/verify_claims.sh <母本.md> <claims.tsv>`（claims.tsv 每行 `claim-id<TAB>逐字引文`）。腳本分三態：✅逐字命中 / ⚠️去空白才中（母本吞空格，回溯失效須修母本）/ 🔴查無（疑似捏造或定位幻覺）。
> **Strict / refuse 門檻**：凡 🔴MISS 的論斷**一律不得寫入最終結論**（腳本 exit 2）；⚠️吞空格須先修母本再驗（exit 3）；全 ✅ 才放行（exit 0）。C5 適配性無法純機械驗證，須在全量 grep 通過後，逐條人工確認「引文真的支撐論斷」。

## 第 4 步 — 架構圖

把「**理論來源(L1) ↔ 本文理論架構(L2) ↔ 研究方法 ↔ 研究發現**」整合成 Mermaid `graph TD`（分 subgraph 四區），並附 **邊—證據對照表**（每條理論邊 ↔ relation 詞彙 ↔ 逐字引文 ↔ 母本定位 ↔ evidence_type）。
- 圖中**每條理論關係邊**都要能回溯到第 3 步的引文；研究發現節點來自摘要的本文結果。圖**不引入**未驗證的新主張。
- 用 `validate_and_render_mermaid_diagram` 驗證語法（label 含 `&` 須在引號內）。
- 標籤誠實：這是「本文理論架構圖」（基於 citation context），**不是**「已驗證的原典理論關係圖」。

## 第 5 步 — Cross-AI 交叉檢核（選用，預設不自動跑）

對**高風險論斷**請外部模型做獨立第二意見，攔下單一模型自洽但錯誤的結論。預設不做；使用者要求、或自評信心不足／論斷風險高時才跑。

**送什麼**（只挑會傷及可信度的三類，不是全文重看）：
- **理論歸屬**：L1 把某概念歸給某文獻、L3 的 relation 詞彙，是否過度詮釋或張冠李戴。
- **引用真偽**：第2步查證結果（DOI/期刊/撤稿）是否有漏判或誤判。
- **關係邊**：架構圖每條理論邊的 relation 是否被升級成語意更強的詞（呼應 C4）。

**怎麼跑**：呼叫 `cross-ai-check`（「事實與文獻查證」或「通用推理檢核」模式），把上述論斷＋對應**逐字引文**一起送出，讓外部模型在有原文脈絡下檢核。

**邊界（關鍵）**：外部模型**也會幻覺**，它的角色是「找分歧點」，不是「裁判真理」。
- 分歧處標 ⚠️，**回母本逐字引文／重跑查證來裁決**，不盲信外部模型翻案。
- 維持本 skill 鐵律：cross-AI 檢核的是「本文如何用文獻／是否過度詮釋」，**不**用來宣稱「原典實際主張什麼」。
- 回報須揭露：跑了哪幾類、幾條、分歧幾條、各以什麼依據裁決。

> 為何重要：C1–C5 防的是「無中生有」「過度詮釋」與「引文—論斷錯配」；cross-AI 多防一層「自洽但整體誤判」——但若把外部模型當權威，等於引入新幻覺源，所以裁決權永遠留在母本。

## 收尾回報

**先結論後細節**：APA7、摘要要點、查證統計（✅/❓/❌ 各幾筆＋分母，含撤稿 🚩）、全文書目體檢結果（⚠️ 命中筆數或「未見明顯瑕疵」）、可疑書目、理論骨架一句話、**claim 全量驗證統計（verify_claims：✅命中 N / ⚠️吞空格 N / 🔴MISS N，strict 是否通過）**、架構圖位置；若跑了第5步，附 cross-AI 分歧清單與裁決依據。

> **失敗拒絕策略（strict）**：若 verify_claims 出現 🔴MISS、或母本品質未達門檻、或核心文獻查無，**不得**照常產出「看似完整」的報告——須降級標紅、明確指出哪幾條不可信並排除於最終結論，或在無法修復時停止，而非掩蓋。

## Forbidden Actions（負面表列）

1. 把論斷寫成「某文獻主張 X」——只能寫「本文用某文獻作為 [角色]／本文如何與之對話」。
2. 把存在性查證當成理論主張正確的證據。
3. 描述理論關係/引用脈絡時不附本文逐字引文，或附了引文卻無法在母本字串搜尋到（違反 C1/C3）。
4. 把分析者歸納的關係或角色標籤標成 `explicit`、或冒充本文原話（違反 C2）——歸納一律標 `interpreted`。
5. 用受控集合以外的關係詞彙，或把本文弱動詞升級成更強的詞（違反 C4，例：control for → 調和）。
6. 公式類主張假裝有逐字依據（母本擷取不全時）——須標「公式擷取受限」。
7. 未經查證對 reference 標「已驗證」；捏造 DOI／期刊分級——查不到一律 `unknown`。
8. 宣稱「已查證全部參考文獻」而實際只查核心子集——必須揭露分母。
9. 把本文架構圖命名為「已驗證理論關係圖 / Confirmed Theory Graph」——它基於 citation context，須誠實標示。
10. 把第5步外部模型的意見當權威直接翻案——cross-AI 只用來標分歧，裁決必須回母本逐字引文／重查，不得盲信外部模型。
11. 把 `verify_claims.sh` 標 🔴MISS（逐字＋去空白皆查無）的論斷寫進最終結論——strict 門檻下必須排除或降級標紅（違反 C3/P0-2）。
12. 引文 grep 得到就當論斷成立，卻不確認引文**實際支撐**該 relation——引文支持的是相鄰句/背景/他人觀點/被反對立場時即為適配失敗，不得照寫（違反 C5）；亦不得把帶 however/not/may 的讓步句抹平成肯定主張。
